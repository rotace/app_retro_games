# Research: Cargo Workspaceと基本ランナーのセットアップ

## 調査結果

### 1. Workspace構造の決定
- **決定**: ADR 0002に従い、プロジェクトルート直下に `core`, `runner-debian`, `runner-wslg` を配置するCargo Workspace構成を採用する。
- **根拠**: 憲法に定義された「物理パッケージの分離」および「厳格な依存方向」をビルドレベルで保証するため。
- **検討した代替案**: 
    - 単一クレート内でのモジュール分割: コア層の物理的な分離が不十分であり、誤った依存関係が混入するリスクがあるため却下。

### 2. 基本実装の検証手法
- **決定**: 各ランナーの `src/main.rs` に標準出力への文字列出力のみを実装し、`cargo run -p <crate>` での動作確認を行う。
- **根拠**: 複雑な依存関係を導入する前に、Workspaceの構成（メンバー定義、パス依存）が正しく機能しているかを確認することが最優先であるため。

### 3. Rustバージョンとツールチェーン
- **決定**: 現在のStableツールチェーンを使用する。
- **根拠**: 特別な機能（nightly-only等）を必要とせず、汎用性と安定性を優先するため。

## 結論
ADR 0002に基づく構成は標準的なRust Workspaceの作法に従っており、技術的なリスクは低い。速やかに設計および実装に移行可能。
