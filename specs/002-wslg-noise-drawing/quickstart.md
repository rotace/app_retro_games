# Quickstart: WSLgノイズ描画検証ガイド
 
## 1. 検証目的
WSLg環境において、`core`層で生成されたパーリンノイズが、`runner-wslg`を通じて正しくウィンドウに表示されることを確認し、描画パイプラインの健全性を検証する。
 
## 2. 事前準備
- WSL2 および WSLg が有効な環境であること。
- 必要な Rust ツールチェーンがインストールされていること。
- 依存ライブラリ（`minifb` 等）のビルドに必要なシステムパッケージがインストールされていること。
 
## 3. 実行手順
 
### 実行コマンド
```bash
cargo run -p runner-wslg
```
 
### 期待される結果
1. WSLg 上に新しいウィンドウが表示される。
2. ウィンドウ内に、滑らかなグラデーションを持つパーリンノイズパターンが描画されている。
3. アプリケーションがフリーズせず、正常に動作し続ける。
 
## 4. 確認ポイント
- [ ] ウィンドウが表示されるか。
- [ ] 描画内容が単純な単色ではなく、ノイズパターンとして認識できるか。
- [ ] 画面に乱れ（アーティファクト）が発生していないか。
