# Quickstart Validation Guide: WSLgノイズ描画機能

## 検証目的
WSLg環境において、パーリンノイズが正しく生成され、ウィンドウ上に表示されることを確認する。

## 前提条件
- WSL2 および WSLg が有効な環境であること。
- Rust ツールチェーンがインストールされていること。

## 検証シナリオ: 基本的なノイズの描画

### 実行コマンド
統合テストとして実装されているため、以下のコマンドで実行する。

```bash
cargo test -p runner-wslg -- --nocapture
```

### 期待される結果
1. WSLg のディスプレイ上に新しいウィンドウが表示される。
2. ウィンドウ内に、静的なパーリンノイズパターン（滑らかなランダム模様）が描画されている。
3. テストが正常に終了し、`ok` が表示される。

## トラブルシューティング
- **ウィンドウが表示されない**: WSLg が正しく動作しているか、`xeyes` 等の単純なX11アプリで確認してください。
- **コンパイルエラー**: `Cargo.toml` の依存関係が正しく設定されているか確認してください。
